# Staff Help Bot ("FAQ Pintar") Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a zero-backend in-app help widget — a floating "?" button opening a keyword-searchable FAQ (Malay) covering leave guides, submission troubleshooting, approval routing, and account help, with light personalization from client data.

**Architecture:** Self-contained module appended to `src/main.js` (vanilla-JS PWA pattern): a `HELP_FAQ` data array + `helpSearch()` pure function + `renderHelpWidget()` DOM widget mounted as a fixed "?" button. No Firestore, no network, no LLM. Two FAQ answers are functions personalized with the in-memory `user`.

**Tech Stack:** Vanilla JS, Vite, Firebase Hosting. No test framework — verify with a Node oracle for `helpSearch`, `node --check`, and Playwright for the UI.

**Spec:** `docs/superpowers/specs/2026-06-08-help-bot-design.md`

**Ground rules:** edit `src/main.js` (and `src/style.css` only if a step says so); `node --check src/main.js` after each task; commit per task; do NOT deploy until Task 4. Temp scripts go in repo root and are deleted before commit.

function helpSearch(query) {
  const q = (query || '').toLowerCase().trim();
  if (!q) return HELP_FAQ.filter(e => e.popular);
  const tokens = q.split(/[^a-z0-9]+/).filter(t => t.length >= 2);
  const exp = new Set(tokens);
  tokens.forEach(t => { if (HELP_SYNONYMS[t]) HELP_SYNONYMS[t].split(' ').forEach(s => exp.add(s)); });
  return HELP_FAQ.map(e => {
    const hay = (e.keywords.join(' ') + ' ' + e.q).toLowerCase();
    let sc = 0;
    exp.forEach(t => { if (e.keywords.includes(t)) sc += 3; else if (hay.includes(t)) sc += 1; });
    return { e, sc };
  }).filter(x => x.sc > 0).sort((a, b) => b.sc - a.sc).slice(0, 6).map(x => x.e);
}
